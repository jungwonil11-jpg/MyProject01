다른 컴퓨터에서 자습 시작 전 실행. study 브랜치를 최신 상태로 맞춤.

## 처음 셋업 (새 컴퓨터에서 딱 한 번만)

```bash
git fetch origin
git checkout -b study origin/study
```

이후로는 아래 실행 순서대로.

## 실행 순서

1. study 브랜치인지 확인 (아니면 `git checkout study`)

2. 로컬 변경사항 확인
   - 있으면: `git stash push --include-untracked -m "시작 전 임시저장"`
   - 없으면: stash 생략

3. origin/study pull: `git pull origin study`

4. stash 했으면 복원: `git stash pop`

5. 충돌 발생 시 사용자에게 알리고 해결 방향 물어봄

6. 완료 요약:
   - 새로 땡겨온 변경사항 목록
   - 현재 git status
