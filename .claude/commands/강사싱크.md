수업 끝난 직후 실행. 강사 코드를 main에 미러링하고, study에 머지함.

## 강사 GitHub 주소
https://github.com/nohssam/2026-springboot01

## 실행 순서

1. 현재 브랜치가 study인지 확인 (아니면 `git checkout study`)

2. study에 변경사항 있으면 commit 또는 stash 선택을 사용자에게 물어볼 것
   - 없으면 생략

3. main으로 이동: `git checkout main`

4. 강사 repo fetch: `git fetch upstream`

5. upstream/main 파일을 main에 그대로 덮어쓰기 (file checkout 방식, unrelated histories 회피):
   `git ls-tree -r upstream/main --name-only | xargs git checkout upstream/main --`

6. main에 commit: `git add -A && git commit -m "sync: 강사 코드 (YYYY-MM-DD)"`
   - 변경사항 없으면 commit 생략

7. main을 origin에 push: `git push origin main`

8. study로 돌아가기: `git checkout study`

9. main을 study에 머지: `git merge main`
   - 충돌 발생 시:
     * 강사가 수정한 부분 살림
     * 내 주석/실습 코드 유지
     * `.claude/`, `CLAUDE.md`, `CLAUDE.local.md` 는 반드시 study 버전 유지

10. stash 했으면 복원: `git stash pop`

11. study를 origin에 push: `git push origin study`

12. 완료 요약:
    - 강사가 추가/변경한 파일 목록
    - 충돌 해결 여부
    - 현재 브랜치 (study) 확인
